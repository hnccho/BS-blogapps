RSS And Atom In Action - Chapter 7 ROME parsing examples

Simple newsfeed parser examples for variety of RSS and Atom newsfeeds.

Build the examples

First, use Ant to build the examples, then cd to the ch07/dist/rome-examples directory to run the examples using the run scripts described later in this document.

For example on UNIX: 
   $ cd $BLOGAPPS\java
   $ ant
   $ cd ch07/dist/rome-examples
Or on Windows: 
   > cd %BLOGAPPS%\java
   > ant
   > cd ch07\dist\rome-examples

Run the examples

To run the examples, change directories into the dist/rome-examples directory and use the run.bat (for Windows) or the run.sh (for UNIX) to the example classes. The run script puts the right classes on the classpath and calls the example class that you specify on the command-line. The examples all take one argument, which must be the path to a newsfeed file on your hard-disk.

com.manning.blogapps.chapter07.GenerateFeedAtom
   Generate Atom via Atom WireFeed model

To run any of the above examples, use the run script. Note that almost all of the above examples require a filename, not a URL. If you want to parse a feed from the web, you'll have to save it to disk first.

For example on UNIX: 
   $ cd $BLOGAPPS/java/ch07/dist/rome-examples
   $ ./run.sh com.manning.blogapps.chapter07.ParseFeed ../../../../testdata/rss_2.0-rssweather.xml
Or on Windows: 
   > cd %BLOGAPPS%\java\ch05\dist\rome-examples
   > run com.manning.blogapps.chapter07.ParseFeed ..\..\..\..\testdata/rss_2.0-rssweather.xml

There are some test newsfeed files that you can try in the $BLOGAPPS/testdata directory.

There are also some example classes that you cannot run: a Atom protocol extensions module and a date-smart RSS 0.91 parser. These are included in the run scripts via src/rome.properties.

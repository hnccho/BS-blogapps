RSS And Atom In Action - Chapter 5 parsing examples

Simple newsfeed parser examples for variety of RSS and Atom newsfeeds.

Build the examples

First, use Ant to build the examples to the dist/parsers directory

For example on UNIX: 
   $ cd $BLOGAPPS\java
   $ ant
Or on Windows: 
   > cd %BLOGAPPS%\java
   > ant

Run the examples

To run the examples, change directories into the dist/parsers directory and use the run.bat (for Windows) or the run.sh (for UNIX) to the example classes. The run script puts the right classes on the classpath and calls the example class that you specify on the command-line. The examples all take one argument, which must be the path to a newsfeed file on your hard-disk.

   com.manning.blogapps.chapter05.ParseRSS10    - Parses RSS 1.0 with JDOM
   com.manning.blogapps.chapter05.ParseRSS20    - Parses RSS 2.0 with JDOM 
   com.manning.blogapps.chapter05.ParseRSSFunky - Parses funky RSS 2.0 with JDOM
   com.manning.blogapps.chapter05.ParseAtom     - Parses Atom with JDOM
   com.manning.blogapps.chapter05.ParseAnyFeed  - Parses any feed with JDOM
   com.manning.blogapps.chapter05.ParseROME     - Parses any feed with ROME

To run any of these, use the run script. Note that the above examples all require a file, not a URL. So if you want to parse a feed from the web, you'll have to save it to disk first.

For example on UNIX: 
   $ cd $BLOGAPPS/java/ch05/dist/parsers
   $ ./run.sh com.manning.blogapps.chapter05.ParseRSS10 example-rss10.xml
Or on Windows: 
   > cd %BLOGAPPS%\java\ch05\dist\parsers
   > run com.manning.blogapps.chapter05.ParseRSS10 example-rss10.xml
There are some test newsfeed files that you can try in the $BLOGAPPS/testdata directory.
